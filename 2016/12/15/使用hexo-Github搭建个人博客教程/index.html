什么是Hexo？

Hexo 是一个简单的博客形态的静态站点生产机器。它有一个模版目录，其中包含原始文本格式的文档，通过 Markdown （或者 Textile） 以及 Liquid 转化成一个完整的可发布的静态网站，你可以发布在任何你喜爱的服务器上。Jekyll 也可以运行在 GitHub Page 上，也就是说，你可以使用 GitHub 的服务来搭建你的项目页面、博客或者网站，而且是完全免费的

安装前提

安装Hexo很简单。在安装前你需要一下应用程序

  • Node.js
  • Git

如果你电脑里面有这些程序，你只需要使用npm就可以安装Hexo了。

安装git

  • windows：安装git
  • Linux(Ubuntu,Debian) $ sudo apt install git-core

安装Node.js

  • windows：到nodejs官网下载并安装
  • linux：这里提供hexo.io官网提供的安装方式就是使用nvm。

安装Hexo

在git和nodejs安装完成后就可以使用，npm安装hexo了。windows、linux安装方式相同

使用自己域名来解析

DNS解析

  1. 先ping下your_user_name.github.io的地址，然后添加两条A记录@，www，如我的域名k3vi.xyz，无论你是访问k3vi.xyz还是访问www.k3vi.xyz都可以访问。此时你可以先访问下看看是不是会报错。
  2. 要想github解析成功就需要在你的本地的网站目录下的source文件夹中新建一个名为CNAME的文件，文件内容为你的域名。之后执行hexo deploy将本地的内容推送到github上面就可以访问了。
    为什么要把文件放在source下面呢？CNAME文件只有放在网站跟目录，你访问你的域名时github才会给你解析，但是hexo每次推送前都会执行hexo clean会把CNAME文件删除，只有放在source文件下面，hexo会自动生成一个CNAME文件到根目录。

评论功能

目前评论系统支持的有多说、Disqus

申请多说

  1. 访问多说官网,使用你的QQ、微博等账号登陆，在官网点击我要安装
  2. 站点地址填写你的域名，多说域名是一会要用到的短名
  3. 查看你的主题是否支持多说，如果支持找到主题目录的下面的_config.yml，例如你的主题是yelee，那么你的_config.yml就在/themes/yelee/_config.yml
  4. 不同的主题在评论功能方面的设置是不同的，支持多说的主题只需要填入你申请的短域名就可以，例如liming.duoshuo.com你只需要填入liming就可以了

写博客

windows用户推荐使用markdown pad 2
新建博文 $ hexo new title
然后在你博客目录source/_post/下面找到你新建的文件，使用markdown pad2编辑title.md,根据markdown语法和hexo语句来编写。具体语法可以参考Hexo官网和markdown语法
